# 平台开盘价接入指示 4

## Codex 已完成的修复

WebUI 的“开盘价 / Price to Beat”已经改为优先读取 Polymarket 页面同源接口：

`https://polymarket.com/api/crypto/crypto-price`

参数：

- `symbol=BTC`
- `eventStartTime=窗口开始 UTC 时间`
- `variant=fiveminute`
- `endDate=窗口结束 UTC 时间`

返回示例：

```json
{
  "openPrice": 77707.50169448124,
  "closePrice": null,
  "completed": false,
  "incomplete": true,
  "cached": true
}
```

WebUI 现在使用 `openPrice` 作为当前窗口开盘价，字段来源标记为：

`ptb_source = polymarket_crypto_price_api`

## 为什么这样改

采集器当前 RTDS 价格已经断流，`price_ticks.jsonl` 停在旧时间。如果继续用采集器里的 `ptb`，新市场会错误沿用旧开盘价。

因此：

1. WebUI 展示层必须优先使用 Polymarket 页面同源 API 的 `openPrice`。
2. 采集器自己的 `ptb` 只能作为 `raw_collector_ptb` 保留用于诊断。
3. 当 Polymarket API 没有返回 `openPrice` 时，不准拿旧 RTDS 价格硬填新窗口开盘价。
4. 如果 `openPrice` 暂时缺失，应显示“开盘价待平台返回 / missing”，并设置 `exclude_from_backtest=true`。

## 不要覆盖的文件

不要回退以下文件中的改动：

- `btc5m-webui/api_server.py`

关键函数：

- `platform_crypto_price()`
- `true_market_snapshot()`

## 仍需修复的根因

这个修复只是保证 WebUI 开盘价和平台页面一致，不代表采集器已经完全正确。

Hermes 仍需修复：

1. RTDS 官方 Chainlink 价格断流问题。
2. `price_ticks.jsonl` 必须持续增长。
3. `rtds_chainlink_online` 不能在价格断流时继续显示在线。
4. 回测数据不能使用外部交易所价格。

## 验收标准

1. WebUI 当前市场开盘价必须等于 Polymarket 页面同源接口 `openPrice`。
2. 新 5 分钟市场切换后，开盘价必须同步变成新窗口的 `openPrice`。
3. 如果平台接口尚未返回 `openPrice`，WebUI 必须显示缺失或待返回，不能沿用上一窗口价格。
4. `raw_collector_ptb` 和 `platform_ptb` 不一致时，以 `platform_ptb` 展示，采集器值仅用于诊断。
