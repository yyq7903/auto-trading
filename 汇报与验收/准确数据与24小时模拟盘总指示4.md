## 必须修复 2：胜负判断规则

Polymarket BTC Up/Down 5m 规则是：

1. 最终价大于或等于开盘价，Up 赢。
2. 最终价小于开盘价，Down 赢。

因此代码里不能继续写：

```python
btc_final > ptb
```

作为 Up 的唯一条件。必须改成：

```python
actual_winner = "Up" if btc_final >= open_price else "Down"
```

注意大小写要统一。WebUI 可以显示 `Up / Down`，文件内部也要尽量统一，避免 `up`、`UP`、`Up` 混用导致统计漏算。
