## 问题根因

旧代码中 `switch_market()` 在 PTB 不可用时返回 `False`，导致 `market_check_loop()` 停留在旧窗口继续写数据。

---

## 已完成修改

### 1. 重写 `switch_market()` — 墙钟时间驱动

- **不再因 PTB 缺失而拒绝切换**
- PTB 只影响质量标记，不能阻止切换
- 创建"临时窗口状态"：`ptb_pending=true`, `current_window_quality=degraded`
- 后台重试获取 PTB（最多 60 秒）

### 2. 重写 `market_check_loop()` — 强制切换逻辑

```python
# 核心：墙钟时间决定是否需要切换
if current_slug == "":
    need_switch = True  # 首次启动
elif slug != current_slug:
    need_switch = True  # 墙钟计算出的窗口与当前不同
elif now >= window_end_ts and window_end_ts > 0:
    need_switch = True  # 当前窗口已过期
```

### 3. 重写 `data_quality_loop()` — 添加健康字段

新增字段：
- `expected_window_start_ts`
- `current_window_start_ts`
- `last_successful_market_switch_at`
- `switch_lag_seconds`
- `negative_seconds_seen`
- `last_price_tick_at`
- `last_orderbook_tick_at`
- `market_switch_reason`
- `ptb_pending`

质量判定逻辑：
- `remaining_seconds < 0` → `stale`
- `ptb_pending` → `degraded`
- 其他根据 WS 连接状态和 tick 数量判定

### 4. 修复 `api_server.py`

- `latest_true_window()` 不再要求 `ptb` 必须存在
- 修复 BOM 字符问题
