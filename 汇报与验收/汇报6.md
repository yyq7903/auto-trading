### 必修 2: 启动模拟按钮必须真正启动服务

**问题**: `/api/toggle` 只改配置，不启动 systemd 服务

**修复**: 在 `toggle()` 函数中添加 systemd 服务启动/停止逻辑
```python
service_name = f"btc5m-{mode}.service"
service_action = "stop" if act == "pause" else "start"
result = subprocess.run(["systemctl", "--user", service_action, service_name], ...)
```

**返回字段**:
- `service_started`: 服务是否启动成功
- `service_active`: 服务是否处于 active 状态
- `service_error`: 启动失败时的错误信息

### 必修 3: WebUI 交易记录 API 必须兼容新字段

**问题**: `/api/trades` 仍按旧字段读取

**修复**: 更新 `do_trades()` 函数，兼容新旧字段
```python
# 兼容新旧字段
amt = float(t.get("amount", t.get("buy_amount", 0)))
pnl = float(t.get("pnl", t.get("net_profit", 0)))
btc_open = t.get("open_price", t.get("platform_open_price", t.get("ptb", 0)))
btc_final = t.get("btc_final", t.get("platform_close_price", None))
```

**新增返回字段**:
- `market_slug`: 市场标识
- `window_start_ts` / `window_end_ts`: 窗口时间
- `market_time`: 市场时间（可读格式）
- `platform_open_price` / `platform_close_price`: 平台价格
- `settlement_status`: 结算状态
- `settle_source`: 结算价来源
- `exclude_from_backtest`: 是否排除回测

### 必修 4: WebUI 交易表前端必须显示新状态

**状态映射**:
- `settlement_status=pending` → 显示 "待平台结算"

- `settlement_status=confirmed` → 显示平台 closePrice
- `status` 区分: `won` / `lost` / `pending` / `skipped` / `failed`
