## 已完成修改

### 1. 修复 PTB 质量检查太严格

**根因**：采集器重启后 RTDS 缓存只有最近几分钟的 tick，lag=-1185000ms（~20分钟）导致切换失败

**修复**：
- `select_ptb_from_rtds()` 增加备选逻辑：如果没有 window_start 之前的 tick，找 window_start 后 5 分钟内的最早 tick
- 质量判定增加 `estimated` 级别：lag 在 -30s ~ +5min 内可接受
- 超过 5 分钟才标记为 `bad`

### 2. 修复空盘口覆盖

**根因**：REST 返回空时直接写入 0，覆盖了之前的有效数据

**修复**：
- 添加 `last_good_orderbook` 全局缓存
- `save_orderbook_snapshot()` 优先级：
  1. REST book 一档价
  2. WebSocket 缓存 (`last_up_bid` 等)
  3. `last_good_orderbook` 缓存
  4. 全部为 0 时不写入

### 3. 强制窗口切换

**修复**：`market_check_loop()` 增加检查：
```python
if current_slug and now >= window_end_ts:
    # 强制切换到新窗口
```

### 4. 新增数据字段

`windows.jsonl` 现在包含：

```json

  "ptb_quality": "estimated",
  "ptb_lag_ms": -15000,
  "platform_ptb": 77003.57,
  "ptb_mismatch": false,
  "exclude_from_backtest": false
}
```
