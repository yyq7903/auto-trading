## 核心问题

旧代码中 `trader.py` 使用 `get_btc_fresh()` 获取 `btc_final`，这是从 Chainlink RPC 获取的本地价格，不是 Polymarket 平台的官方结算价。这会导致：
1. 交易记录、胜率、回测全部偏差
2. 模拟盘结果不可信

## 修复详情

### 修复 1：结算价来源

**旧代码**:
```python
btc_final = get_btc_fresh(retries=3)  # 从 Chainlink RPC 获取
```

**新代码**:
```python
platform_price = fetch_platform_crypto_price(window_start_ts, window_end_ts)
btc_final = platform_price.get("closePrice")  # 从 Polymarket 平台获取
```

### 修复 2：胜负判断规则

**旧代码**:
```python
won = (direction == "Up" and btc_final > ptb) or (direction == "Down" and btc_final < ptb)
```

**新代码**:
```python
actual_winner = "Up" if btc_final >= open_price else "Down"
won = (direction == actual_winner)
```

**注意**: 使用 `>=` 而不是 `>`，因为 Polymarket 规则是"最终价大于或等于开盘价，Up 赢"。

### 修复 3：结算状态处理

**新增逻辑**:
```python
if btc_final is None or btc_final <= 0:
    # 平台结算价未返回，标记为待结算
    settlement_status = "pending"
    exclude_from_backtest = True
else:
    # 平台结算价已返回
    settlement_status = "confirmed"
    exclude_from_backtest = False
```

### 修复 4：Trade Record 格式

**新增字段**:
- `market_slug`: 市场标识
- `window_start_ts`: 窗口开始时间戳
- `window_end_ts`: 窗口结束时间戳
- `market_time`: 市场时间（可读格式）
- `open_price`: 开盘价
- `platform_open_price`: 平台开盘价
- `platform_close_price`: 平台结算价
- `settlement_gap`: 结算价差
- `settlement_status`: 结算状态
- `settle_source`: 结算价来源
- `settle_confirmed_at`: 结算确认时间
- `exclude_from_backtest`: 是否排除回测
