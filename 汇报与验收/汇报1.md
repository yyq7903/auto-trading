## 问题

WebUI 显示的 Price to Beat (PTB) 与平台实际值偏差 $26+：

```
平台 PTB：$77,343.67
WebUI PTB：$77,369.98
```

**根因**：旧代码用 `last_rtds_price`（当前最新 RTDS 价格）当 PTB，但市场切换时 RTDS 价格可能已是窗口开始后的价格，导致几十美元偏差。

---

## 已完成修改

### 1. 添加 RTDS tick 环形缓存

```python
rtds_tick_buffer = deque(maxlen=1200)  # 10分钟，每秒1条
```

每次收到 RTDS Chainlink tick 都写入缓存。

### 2. 新增 `select_ptb_from_rtds(window_start_ms)`

从缓存中反查 `timestamp_ms <= window_start_ms` 的最接近 tick。

### 3. 重写 `switch_market()` PTB 逻辑

- **删除**：`last_rtds_price` fallback
- **删除**：CLOB ask fallback（那是概率，不是 BTC 价格）
- **新增**：质量标记（exact/close/bad）
- **新增**：无 tick 时等待，不创建正式窗口

### 4. 新增 `fetch_platform_ptb(slug)` + `validate_ptb_async()`

后台异步抓取平台页面 PTB，与本地 PTB 对比：
- `diff ≤ $1.00` → 校验通过
- `diff > $1.00` → 标记 `ptb_mismatch=True, exclude_from_backtest=True`

### 5. 新增数据字段

`windows.jsonl` 现在包含：

```json
{
  "window_start_ms": 1779799500000,
  "ptb": 77014.10,
  "ptb_source": "polymarket_rtds_chainlink",
  "ptb_timestamp_ms": 1779799500000,
  "ptb_lag_ms": 0,
  "ptb_quality": "exact",
  "platform_ptb": 77014.10,
  "ptb_mismatch": false,
  "exclude_from_backtest": false
}
```
